# Путешествия по России

Многостраничный учебный веб‑сайт о путешествиях по России, созданный с использованием HTML, CSS (Flexbox и Grid) и JavaScript. На сайте реализованы адаптивная верстка, интерактивная карусель маршрутов и обработка формы обратной связи.

## Структура проекта

```
lab4/
├── index.html              # Главная страница (Flexbox-верстка, hero, highlights, stories)
├── destinations.html       # Маршруты (CSS Grid → интерактивная карусель на JS)
├── feedback.html           # Обратная связь (форма с валидацией на JS)
├── css/
│   └── style.css           # Общие стили, Flexbox, Grid, карусель, адаптивность
├── js/
│   └── app.js              # Логика навигации, карусели и обработки формы
├── images/                 # Изображения для страниц
│   ├── russia-landscape.jpg
│   ├── red-square.jpg
│   ├── hermitage.jpg
│   ├── baikal.jpg
│   ├── peterhof.jpg
│   ├── baikal_1.jpg
│   └── baikal_2.jpg
└── README.md               # Отчет по проделанной работе

```

### Используемые технологии

HTML5: семантическая разметка, многостраничная структура сайта.

CSS3: Flexbox и Grid для верстки, адаптивный дизайн, анимации и эффекты.​

JavaScript: работа с DOM, обработка событий, интерактивная карусель, логика формы и навигации.​



#### Реализованные функции:

1.Навигация и активное меню
	Подсветка текущей страницы в шапке:

		Определение текущего пути window.location.pathname.

		Добавление класса .is-active соответствующей ссылке навигации.

		Улучшает ориентацию пользователя между страницами сайта.​

	Мобильное меню:

		Кнопка «Меню» показывает/скрывает список ссылок.

		Класс .is-open управляет видимостью блока .site-nav__links.

		Используются ARIA-атрибуты aria-expanded и aria-controls для доступности.



2. Главная страница (index.html)
	Блок hero:

		Flexbox-верстка для текста и изображения.

		Анимация «плавающей» картинки с помощью @keyframes float.

		Две CTA‑кнопки: переход к маршрутам и форме обратной связи.

	Блоки «Почему TravelRu?» и «Истории путешественников»:

		Карточки на Flexbox, адаптивный расклад под разные ширины экрана.

		Используются тени, скругления и CSS‑переменные для единого стиля.

3. Страница маршрутов (destinations.html)
	Карусель маршрутов:

		Горизонтальный трек .carousel-track на Flexbox с плавным transform: translateX(...).

		Одновременно отображается 3 карточки на десктопе, 2 на планшетах и 1 на мобильных.

		Добавлены кнопки навигации «❮» и «❯», которые листают карточки влево/вправо.

		Индикаторы‑точки показывают текущую позицию карусели и позволяют перейти к нужному слайду.

		Клавиатурная навигация: стрелки ← и → управляют каруселью.​

	Анимация появления карточек:

		Каждая карточка .carousel-item анимируется через @keyframes slideInUp.

		Используются разные animation-delay для каскадного появления элементов.

		Создает современный и живой интерфейс при загрузке страницы.​

4. Страница обратной связи (feedback.html)
	Адаптивная форма:

		Используется CSS Grid (.feedback-form) для раскладки полей.

		Поддерживаются три типа полей: текстовые, select и textarea, а также чекбокс согласия.

		Поля выстраиваются в несколько колонок на большом экране и в одну колонку на мобильных.

	Обработка формы в JavaScript:

		Класс Submission инкапсулирует данные формы (имя, email, стиль, комментарий, согласие, время отправки).

		При отправке:

			Выполняется event.preventDefault() без перезагрузки страницы.

			Данные собираются через FormData, приводятся к нужному виду (trim и булевые значения).

			В консоль выводится форматированный отчет с данными и временем отправки.

			Форма сбрасывается, пользователю показывается уведомление alert.

5. Общая логика и улучшения
	Автоматическое обновление года в футере:

		Скрипт находит <span id="year"> и подставляет текущий год.

		Исключает необходимость ручного редактирования при смене года.




#### Технические детали:

Весь JavaScript обёрнут в немедленно вызываемое функциональное выражение (IIFE), чтобы не загрязнять глобальную область видимости.

Проверка наличия элементов (if (form) { ... }, if (carouselTrack && prevBtn && nextBtn && indicatorsContainer)) позволяет безопасно использовать один общий app.js для всех страниц.

Для карусели применён подход с вычислением сдвига в процентах (translateX(-offset%)) и управлением количеством видимых карточек через CSS и JS.​

Макет строится комбинацией Flexbox и Grid, что соответствует современным практикам верстки адаптивных интерфейсов.​

## Запуск проекта
Откройте файл index.html в браузере (двойным кликом или через локальный сервер).

Перейдите по ссылке «Маршруты», чтобы увидеть интерактивную карусель направлений.

Перейдите на страницу «Обратная связь», заполните форму и отправьте данные:

Проверьте консоль браузера (F12 → Console), чтобы увидеть структурированный лог отправки.

Изменяйте размер окна браузера, чтобы увидеть адаптивное поведение Flexbox, Grid и карусели.
